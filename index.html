Linux for DevOps
Command line, scripting, system administration & networking
Weeks 1–4 Foundation Track Start First
▼
PHASE 1.1 CLI Basics & Filesystem Week 1
Navigation: ls, cd, pwd, find, locate
File ops: cp, mv, rm, mkdir, touch, ln
Permissions: chmod, chown, umask, ACLs
Text processing: cat, grep, sed, awk, cut
I/O redirection, pipes, stdin/stdout/stderr
Vim / Nano basics — edit files in terminal
Resources
Linux Journey (linuxjourney.com) The Linux Command Line (book) OverTheWire: Bandit (wargame)
🎯Project: Write a bash script that backs up a directory, compresses it, and logs the result with a timestamp.
PHASE 1.2 Process & System Management Week 2
Process management: ps, top, htop, kill, nice
Systemd: systemctl, journalctl, service units
Cron jobs & scheduling
Disk management: df, du, lsblk, mount
Package managers: apt, yum/dnf, snap
Environment variables & .bashrc / .profile
🎯Project: Write a system health-check script (CPU, memory, disk, running services) that emails an alert if thresholds are breached.
PHASE 1.3 Networking & Security Basics Week 3
TCP/IP, DNS, DHCP, HTTP/S, SSH
Tools: ping, traceroute, netstat, ss, curl, wget
iptables / ufw firewall rules
SSH key management, scp, rsync
User/group management, sudo, PAM
File integrity: md5sum, sha256sum
🎯Project: Set up a hardened Ubuntu VM — SSH key-only login, firewall rules, fail2ban, non-root user.
PHASE 1.4 Shell Scripting Mastery Week 4
Bash scripting: variables, arrays, functions
Conditionals, loops, case statements
Exit codes & error handling (set -e, trap)
Script arguments, getopts
Regular expressions in bash
Writing idempotent scripts
Resources
Bash Guide (mywiki.wooledge.org) ShellCheck (linting tool)
🎯Capstone: Full server provisioning script — installs packages, configures users, sets up firewall, deploys a sample app.
🐍
Python for DevOps
Automation, scripting, APIs, infrastructure tooling
Weeks 3–8 Automation Focus
▼
PHASE 2.1 Python Core Fundamentals Weeks 3–4
Data types, variables, operators
Control flow: if/else, for, while, comprehensions
Functions, *args, **kwargs, decorators
OOP: classes, inheritance, dunder methods
File I/O, context managers (with)
Exception handling, logging module
Resources
Automate the Boring Stuff (book) Python for DevOps (O'Reilly book) Real Python tutorials
🎯Project: CLI tool that parses log files, extracts errors, counts occurrences, and generates a report.
PHASE 2.2 DevOps-Specific Python Libraries Weeks 5–6
os, sys, subprocess — system interaction
pathlib, shutil — file system automation
requests, httpx — REST API calls
paramiko, fabric — SSH automation
json, yaml, toml — config parsing
boto3 (AWS SDK), google-cloud SDK basics
🎯Project: Python script that creates AWS EC2 instances, tags them, and reports their status via a Slack webhook.
PHASE 2.3 Testing, Packaging & Automation Weeks 7–8
pytest — unit + integration tests
mocking with unittest.mock
Virtual environments, pip, pyproject.toml
Click / argparse — CLI frameworks
Docker SDK for Python
Type hints & mypy
🎯Capstone: Build a Python-based "mini-Ansible" — reads a YAML config, SSHes into servers, and runs provisioning commands.
☁️
Cloud (AWS / GCP / Azure)
Core services, IaC, networking, compute, storage, IAM
Weeks 5–12 AWS Focus Cert Path
▼
PHASE 3.1 Cloud Concepts & Core Services Weeks 5–6
IaaS vs PaaS vs SaaS, shared responsibility
Regions, AZs, edge locations
EC2: instance types, AMIs, EBS, security groups
S3: buckets, policies, versioning, lifecycle
VPC: subnets, route tables, NAT, IGW
IAM: users, roles, policies, MFA
Resources
AWS Cloud Practitioner (CLF-C02) AWS Skill Builder (free tier) Adrian Cantrill (course)
🎯Project: Deploy a static website on S3 with CloudFront CDN, custom domain, and HTTPS using ACM.
PHASE 3.2 Compute, Containers & Serverless Weeks 7–9
Auto Scaling Groups, Load Balancers (ALB/NLB)
Docker fundamentals: images, containers, Dockerfile
ECS (Fargate) & ECR
Lambda: event triggers, layers, IAM roles
API Gateway + Lambda integration
RDS, DynamoDB, ElastiCache basics
🎯Project: Containerize a Python Flask app → push to ECR → deploy on ECS Fargate behind an ALB.
PHASE 3.3 Infrastructure as Code (Terraform) Weeks 10–11
Terraform HCL: providers, resources, variables
State files, remote backends (S3 + DynamoDB)
Modules, workspaces, outputs
CloudFormation basics (for AWS-native IaC)
Terraform best practices & directory structure
Drift detection, import, taint
Resources
HashiCorp Terraform Tutorials Terraform: Up & Running (book)
🎯Project: Terraform full VPC stack — VPC, subnets, IGW, security groups, EC2 with user-data, S3 backend.
PHASE 3.4 Monitoring, Cost & Security Week 12
CloudWatch metrics, logs, alarms, dashboards
CloudTrail — auditing API calls
AWS Cost Explorer & Budgets
Security Hub, GuardDuty basics
Secrets Manager vs SSM Parameter Store
KMS, encryption at rest/in-transit
🎯Capstone: Full production-like AWS environment via Terraform with CloudWatch monitoring, budget alerts, and IAM least-privilege setup.
💻
CS Fundamentals
Data structures, algorithms, OS concepts, networking internals
Weeks 9–14 Interview Prep
▼
PHASE 4.1 Data Structures Weeks 9–10
Arrays, strings, hash maps / sets
Stacks, queues, deques
Linked lists (singly & doubly)
Trees: BST, heaps, tries
Graphs: adjacency list/matrix
Big-O notation, time/space complexity
Resources
NeetCode.io (roadmap) LeetCode (Easy → Medium) CS50x (Harvard free course)
🎯Milestone: Solve 40 LeetCode problems (arrays, strings, hashing) with Python.
PHASE 4.2 Algorithms Weeks 11–12
Sorting: merge, quick, heap, counting
Searching: binary search, BFS, DFS
Recursion & backtracking
Dynamic programming (memoization, tabulation)
Greedy algorithms
Two pointers, sliding window patterns
🎯Milestone: Solve 30 Medium LeetCode problems covering trees, graphs, and DP.
PHASE 4.3 Operating Systems & Networking Weeks 13–14
Processes vs threads, context switching
Memory: virtual memory, paging, segmentation
Concurrency: mutex, semaphores, deadlocks
OSI model & TCP/IP stack deep dive
HTTP/1.1 vs HTTP/2, TLS handshake
DNS resolution, CDN, reverse proxy
Resources
OSTEP (Operating Systems: Three Easy Pieces) Computer Networking: A Top-Down Approach Hussein Nasser (YouTube)
🎯Capstone: Explain HTTP request lifecycle end-to-end (DNS → TCP → TLS → HTTP → response) — write + present a 1-page explainer.
🏗️
System Design
Scalability, reliability, distributed systems & architecture patterns
Weeks 13–20 Senior-Level
▼
PHASE 5.1 Foundations of Scale Weeks 13–14
Vertical vs horizontal scaling
Load balancing: Round-robin, L4 vs L7
Caching: CPU cache, Redis, CDN, cache invalidation
Database scaling: read replicas, sharding
CAP theorem, eventual consistency
Latency vs throughput, SLA / SLO / SLI
Resources
System Design Primer (GitHub) Designing Data-Intensive Applications (book) ByteByteGo (blog + YouTube)
🎯Exercise: Design a URL shortener (like bit.ly) — cover DB choice, hashing, redirects, analytics at scale.
PHASE 5.2 Distributed Systems Patterns Weeks 15–16
Message queues: Kafka, SQS, RabbitMQ
Microservices vs monolith trade-offs
API design: REST vs GraphQL vs gRPC
Service discovery, circuit breakers
Consistent hashing
Two-phase commit, SAGA pattern
🎯Exercise: Design a notification system for 10M users — push, email, SMS delivery with fan-out.
PHASE 5.3 DevOps-Specific System Design Weeks 17–18
CI/CD pipeline architecture
Blue/green, canary, rolling deployments
Kubernetes architecture: pods, services, ingress
Observability: metrics, logs, traces (3 pillars)
Disaster recovery: RPO vs RTO
Multi-region architectures, failover
Resources
Kubernetes Docs Google SRE Book (free online)
🎯Exercise: Design a CI/CD system for a 100-engineer org — pipelines, artifact storage, rollback strategy.
PHASE 5.4 Full System Design Interviews Weeks 19–20
Design YouTube / Netflix (video streaming)
Design a distributed key-value store
Design a rate limiter
Design a web crawler
Mock interviews & whiteboard practice
Back-of-envelope estimation drills
🎯Capstone: Complete 5 full mock system design sessions (recorded) covering requirements → estimation → design → trade-offs.
